1. Définition d’une classe

Une classe est un modèle pour créer des objets. Elle regroupe des propriétés (variables) et des méthodes (fonctions) qui décrivent un objet.

<?php
class Utilisateur {
    // Propriétés
    public $nom;
    public $email;

    // Méthode constructeur (appelée à la création d'un objet)
    public function __construct($nom, $email) {
        $this->nom = $nom;
        $this->email = $email;
    }

    // Méthode
    public function afficherInfos() {
        return "Nom: $this->nom, Email: $this->email";
    }
}
?>

  • Le mot-clé class permet de définir une nouvelle classe nommée Utilisateur.

  • La méthode spéciale __construct() est appelée lors de la création d’un objet et sert à initialiser les propriétés.

  • $this fait référence à l’objet courant et permet d’accéder aux propriétés et méthodes.

2. Les getters et les setters

Les getters et setters sont des méthodes en programmation orientée objet utilisées pour accéder et modifier les propriétés d’un objet. Ils permettent de gérer la visibilité et la validation des données et assurent une meilleure encapsulation.

class User {
    private $name;

    //Getter
    public function getName() {
        return $this->name;
    }

    //Setter
    public function setName($name) {
            $this->name = $name;
    }
}

Bonnes pratiques :

  • Valider les données avant de les attribuer à la propriété pour éviter les erreurs.

  • Utiliser private pour les propriétés et accédez-y uniquement via les getters et setters.

  • Utiliser des noms de méthodes clairs et cohérents (get et set) suivi du nom de la propriété, ce qui facilite la compréhension.

3. Création d’un objet

Un objet est une instance d’une classe. On peut créer autant d’objets qu’on le souhaite à partir d’une classe.

<?php
// Création d'un nouvel objet à partir de la classe Utilisateur
$utilisateur1 = new Utilisateur("Alice", "alice@ozenne.com");

// Affichage des informations de l'utilisateur
echo $utilisateur1->afficherInfos();
?>

  • Le mot-clé new permet de créer une nouvelle instance de la classe Utilisateur.

  • L’objet $utilisateur1 est créé avec les valeurs passées au constructeur et stockées dans les propriétés nom et email.

4. Héritage

L’héritage permet à une classe de récupérer les propriétés et méthodes d’une autre classe.

<?php
class Admin extends Utilisateur {
    public $role;

    public function __construct($nom, $email, $role) {
        parent::__construct($nom, $email); // Appel au constructeur de la classe parent
        $this->role = $role;
    }

    public function afficherInfos() {
        return parent::afficherInfos() . ", Rôle: $this->role";
    }
}
?>

  • La classe Admin hérite de la classe Utilisateur grâce au mot-clé extends.

  • La méthode parent:: permet d’accéder aux méthodes ou propriétés de la classe parent.

5. Encapsulation

L’encapsulation consiste à protéger les propriétés d’une classe pour empêcher un accès direct. On utilise des méthodes appelées getters et setters pour accéder et modifier ces propriétés.

<?php
class Utilisateur {
    private $nom;
    private $email;

    public function __construct($nom, $email) {
        $this->nom = $nom;
        $this->email = $email;
    }

    public function getNom() {
        return $this->nom;
    }

    public function setNom($nom) {
        $this->nom = $nom;
    }
}
?>

  • Le mot-clé private rend les propriétés nom et email privées, empêchant l’accès direct depuis l’extérieur de la classe.

  • Les méthodes getNom() et setNom() permettent d’accéder et de modifier ces propriétés de manière contrôlée.

6.2. Autre Exemple

6.2.1. Les classes

Classe Animal

La classe Animal représente un animal générique. Elle contient des attributs comme le nom et l'`age`, ainsi qu’une méthode communiquer() qui sera définie dans les classes enfants.

Le mot-clé protected utilisé pour les attributs signifie que ces propriétés ne peuvent pas être accédées directement depuis l’extérieur de la classe, mais elles peuvent être utilisées dans la classe elle-même et dans toutes les classes qui en héritent. Cela permet de protéger les données tout en permettant l’accès aux sous-classes.

<?php
class Animal {
    protected $nom;
    protected $age;

    public function __construct($nom, $age) {
        $this->nom = $nom;
        $this->age = $age;
    }

    public function getNom() {
        return $this->nom;
    }

    public function communiquer() {
        return "L'animal communique.";
    }
}
?>
Classe Chien

La classe Chien hérite de Animal. Elle ajoute un attribut race et une méthode aboyer().

<?php
class Chien extends Animal {
    private $race;

    public function __construct($nom, $age, $race) {
        parent::__construct($nom, $age);
        $this->race = $race;
    }

    public function aboyer() {
        return "Le chien aboie.";
    }
}
?>
Classe Chat

La classe Chat hérite également de Animal. Elle ajoute un attribut race et une méthode miauler().

<?php
class Chat extends Animal {
    private $race;

    public function __construct($nom, $age, $race) {
        parent::__construct($nom, $age);
        $this->race = $race;
    }

    public function miauler() {
        return "Le chat miaule.";
    }
}
?>

6.2.2. Création d’objets dans une page

Nous allons maintenant créer des objets pour les races spécifiques de chiens et de chats : Labrador, Pitbull, Berger pour les chiens, et un chat.

<?php
$labrador = new Chien("Buddy", 3, "Labrador");
$pitbull = new Chien("Rocky", 5, "Pitbull");
$berger = new Chien("Max", 4, "Berger");

$monChat = new Chat("Choupette", 2, "Siamois");

echo $labrador->communiquer(); // Affiche "L'animal communique."
echo $labrador->aboyer();        // Affiche "Le chien aboie."
echo "Le nom du Labrador est " . $labrador->getNom(); // Affiche "Le nom du Labrador est Buddy"

echo $monChat->communiquer();    // Affiche "L'animal communique."
echo $monChat->miauler();          // Affiche "Le chat miaule."
echo "Le nom du Siamois est " . $monChat->getNom(); // Affiche "Le nom du Siamois est Choupette"
?>

Dans cette structure, un Chien est un Animal, et un Chat est également un Animal, mais tous les Animaux ne sont pas des Chiens ou des Chats.
Cette hiérarchie de classes permet de mieux organiser le code et de réutiliser les fonctionnalités communes entre les classes.

7. Polymorphisme

Le polymorphisme permet à des objets de différentes classes d’être traités de manière interchangeable s’ils partagent des méthodes ou implémentent une même interface.

class Animal {
    public function communiquer() {
        return "L'animal fait un bruit.";
    }
}

class Chien extends Animal {
    public function communiquer() {
        return "Le chien aboie.";
    }
}

class Chat extends Animal {
    public function communiquer() {
        return "Le chat miaule.";
    }

}

function faireCommuniquer(Animal $animal) {
    echo $animal->communiquer();
}

faireCommuniquer(new Chien()); // Affiche "Le chien aboie."
faireCommuniquer(new Chat());  // Affiche "Le chat miaule."

La surcharge de méthodes est un élément central du polymorphisme en POO. Elle permet à une sous-classe de fournir une version spécifique d’une méthode qui a été définie dans sa classe parente. C’est ce qui permet d’avoir un comportement différent selon le type d’objet, même si ces objets partagent une méthode avec le même nom.

8. Interfaces

Une interface définit un ensemble de méthodes qu’une classe doit implémenter. Cela permet de définir un comportement commun sans imposer une hiérarchie d’héritage.

  • Toutes les méthodes d’une interface sont abstraites et doivent être implémentées par les classes qui l’implémentent.

  • Une interface ne peut pas avoir de champs (variables d’instance).

  • Elle décrit une relation a-un entre les classes qui l’implémentent.

interface Volant {
    public function voler();
}

class Avion implements Volant {
    public function voler() {
        return "L'avion vole.";
    }
}

class Oiseau implements Volant {
    public function voler() {
        return "L'oiseau bat des ailes.";
    }
}

function faireVoler(Volant $objet) {
    echo $objet->voler();
}

faireVoler(new Avion()); // Affiche "L'avion vole."
faireVoler(new Oiseau()); // Affiche "L'oiseau bat des ailes."

  • Les interfaces permettent de définir des comportements communs que différentes classes peuvent implémenter sans être liées par héritage.

8.1. Implémenter plusieurs interfaces

L’héritage multiple n’est pas autorisé en PHP. Cependant, il est possible de définir des contrats que plusieurs classes peuvent implémenter.
Cela permet à différentes classes de partager des méthodes sans avoir à hériter d’une classe commune.

interface Interface1 {
    public function methode1();
}

interface Interface2 {
    public function methode2();
}

class MaClasse implements Interface1, Interface2 {
    public function methode1() {
        // Implémentation de la méthode de l'interface 1
    }

    public function methode2() {
        // Implémentation de la méthode de l'interface 2
    }
}

9. Abstraction

Les classes abstraites ne peuvent pas être instanciées directement doit être héritée par une autre classe.

  • Elle peut avoir des méthodes abstraites (sans implémentation) ainsi que des méthodes concrètes (avec implémentation).

  • Les classes qui héritent d’une classe abstraite doivent implémenter les méthodes abstraites.

  • Une classe abstraite peut avoir des champs (variables d’instance) et des méthodes non abstraites.

  • Elle décrit une relation est-un entre les classes qui l’héritent.

abstract class Animal {
    protected $nom;

    public function __construct($nom) {
        $this->nom = $nom;
    }

    abstract public function communiquer();
}

class Chien extends Animal {
    public function communiquer() {
        return "Le chien aboie.";
    }
}

class Chat extends Animal {
    public function communiquer() {
        return "Le chat miaule.";
    }
}

$chien = new Chien("Rex");
echo $chien->communiquer();  // Affiche "Le chien aboie."

  • Les classes abstraites définissent des comportements communs tout en forçant les sous-classes à implémenter certaines méthodes.

10. Classe abstraite VS Interface

Classe Abstraite Interface

Héritage multiple

Une classe ne peut hériter que d’une seule classe abstraite.

Une classe peut implémenter plusieurs interfaces.

Méthodes

Peut contenir des méthodes abstraites et des méthodes concrètes.

Ne peut contenir que des méthodes abstraites.

Propriétés

Peut avoir des propriétés.

Ne peut pas avoir de propriétés.

Constructeur

Peut avoir un constructeur.

Ne peut pas avoir de constructeur.

Accès aux membres

Les méthodes peuvent avoir des niveaux d’accès (public, protected, private).

Les méthodes sont forcément public.

Utilisation

Utilisée lorsque vous avez des méthodes communes avec une implémentation partielle et que vous souhaitez partager du code entre plusieurs classes.

Utilisée pour définir des comportements que des classes doivent implémenter, sans imposer d’héritage.

11. Les Statiques en PHP

Contrairement à d’autres langages comme Java ou C#, PHP ne supporte pas la notion de classe entière définie comme statique.

En PHP, seules les propriétés et les méthodes peuvent être déclarées comme statiques. Elles sont utilisées lorsque vous voulez qu’une méthode ou une propriété appartienne à la classe dans son ensemble, et non à chaque objet.

11.1. Propriétés Statiques

Une propriété statique est partagée par tous les objets de la classe. Cela signifie que toutes les instances de cette classe accèdent à la même valeur de la propriété.

<?php
class Compteur {
    public static $compte = 0;

    public static function incrementer() {
        self::$compte++;
    }
}

// Incrémenter le compteur
Compteur::incrementer();
Compteur::incrementer();
echo Compteur::$compte; // Affiche: 2
?>

11.2. Méthodes Statiques

Une méthode statique peut être appelée directement depuis la classe, sans avoir besoin de créer une instance (un objet) de cette classe.

<?php
class MathUtil {
    public static function carre($nombre) {
        return $nombre * $nombre;
    }
}

// Appel de la méthode statique sans créer d'instance de la classe
echo MathUtil::carre(4); // Affiche: 16
?>

  • Elles sont associées à la classe elle-même, pas aux objets (instances).

  • Elles peuvent être appelées directement avec ClassName::method() ou ClassName::$property, sans créer d’instance.

  • Elles sont souvent utilisées pour des utilitaires ou des valeurs partagées par tous les objets d’une classe.

12. Trait (Spécifique à PHP)

Les traits sont utiles lorsqu’on a du code commun à plusieurs classes, mais qu’on ne peut pas utiliser l’héritage, soit parce qu’il n’y a pas de lien logique entre les classes, soit parce qu’une classe est déjà en train d’hériter d’une autre (en PHP, une classe ne peut hériter que d’une seule classe).

<?php
trait Salut {
    public function direBonjour() {
        return "Bonjour !";
    }
}

class Utilisateur {
    use Salut; // Utilise le trait Salut
}

class Admin {
    use Salut; // Utilise le trait Salut
}

$utilisateur = new Utilisateur();
echo $utilisateur->direBonjour(); // Affiche "Bonjour !"

$admin = new Admin();
echo $admin->direBonjour(); // Affiche "Bonjour !"
?>

Dans cet exemple :

  • Le trait Salut contient une méthode direBonjour().

  • Les classes Utilisateur et Admin utilisent le mot-clé use pour inclure le trait Salut. Cela permet à ces deux classes d’utiliser la méthode direBonjour() sans avoir à la redéfinir dans chaque classe.

  • Le trait permet de réutiliser du code dans plusieurs classes sans avoir besoin d’utiliser l’héritage. Les deux classes partagent la même méthode direBonjour() mais ne sont pas liées par un lien d’héritage.

  • Les traits permettent la réutilisation de méthodes dans des classes non liées.

13. Gestion des exceptions

Les exceptions permettent de gérer les erreurs dans le code de manière contrôlée et de capturer des événements inattendus.

class Division {
    public function diviser($a, $b) {
        if ($b == 0) {
            throw new Exception("Division par zéro impossible.");
        }
        return $a / $b;
    }
}

try {
    $division = new Division();
    echo $division->diviser(10, 0);
} catch (Exception $e) {
    echo "Erreur : " . $e->getMessage();
}

  • L’usage des exceptions permet de traiter les erreurs et de réagir aux situations imprévues dans l’exécution du programme.

14. Hydratation des Données

L'hydratation en PHP est une technique de programmation orientée objet qui consiste à remplir les propriétés d’un objet à partir d’un ensemble de données, souvent issues d’une base de données. Elle permet de mapper automatiquement les données sur les propriétés de l’objet, simplifiant ainsi l’initialisation.

Supposons que nous avons une base de données contenant des informations sur des utilisateurs, et que nous voulons créer des objets Utilisateur pour chaque enregistrement.

14.1. Classe Utilisateur

<?php
class Utilisateur {
    private $id;
    private $nom;
    private $prenom;
    private $email;

    // Constructeur qui appelle la méthode hydrate pour initialiser l'objet
    public function __construct(array $data = []) {
        $this->hydrate($data);
    }

    // Méthode d'hydratation
    private function hydrate(array $data) {
        foreach ($data as $key => $value) {
            $method = 'set' . ucfirst($key);
            if (method_exists($this, $method)) {
                $this->$method($value);
            }
        }
    }

    // Getters et setters
    public function getId() {
        return $this->id;
    }

    public function setId($id) {
        $this->id = $id;
    }

    public function getNom() {
        return $this->nom;
    }

    public function setNom($nom) {
        $this->nom = $nom;
    }

}

14.2. Utilisation de l’Hydratation

En créant une nouvelle instance de Utilisateur, on peut directement lui passer les données pour initialiser ses propriétés :

$data = [
    'id' => 1,
    'nom' => 'Dupont',
    'prenom' => 'Jean',
    'email' => 'jean.dupont@example.com'
];

$utilisateur = new Utilisateur($data);
echo $utilisateur->getNom(); // Affiche "Dupont"

15. Glossaire

Concept Caractéristiques

Classe

Une classe est un modèle pour créer des objets. Elle définit les attributs (propriétés) et les méthodes (comportements) que ces objets auront.

Objet

Un objet est une instance d’une classe. Chaque objet possède ses propres valeurs pour les propriétés définies dans la classe et peut exécuter des méthodes.

Héritage

L'héritage permet à une classe d’hériter des attributs et méthodes d’une autre classe (classe parente). Il favorise la réutilisation de code et permet de spécialiser le comportement d’une classe.

Polymorphisme

Le polymorphisme permet d’utiliser des objets de différentes classes de manière interchangeable, à condition qu’ils partagent des méthodes. Il existe deux types :

- Polymorphisme par redéfinition (overriding) : une sous-classe redéfinit une méthode de la classe parent.

- Polymorphisme par surcharge (overloading) : PHP ne supporte pas la surcharge de méthodes par le nombre de paramètres comme dans d’autres langages, mais cela permet d’utiliser des méthodes du même nom dans différentes classes liées.

Encapsulation

L'encapsulation consiste à protéger les données d’un objet en les rendant accessibles uniquement via des méthodes spécifiques (getters et setters). Cela empêche l’accès direct aux propriétés depuis l’extérieur de la classe.

Abstraction

L'abstraction est utilisée pour définir des classes abstraites ou des méthodes abstraites qui ne sont pas implémentées dans la classe parent, mais qui doivent être implémentées dans les sous-classes. Les classes abstraites ne peuvent pas être instanciées directement.

Interface

Une interface définit un ensemble de méthodes qu’une classe doit implémenter. Contrairement à une classe abstraite, une interface ne contient que des déclarations de méthodes sans implémentation, et une classe peut implémenter plusieurs interfaces.

Trait

Les traits permettent de réutiliser des méthodes dans plusieurs classes sans utiliser l’héritage. Un trait contient des méthodes que plusieurs classes peuvent inclure à l’aide du mot-clé use. Ils sont utiles pour éviter la duplication de code.

Surcharge de méthodes (Overriding)

La surcharge de méthode consiste à redéfinir une méthode héritée dans une sous-classe pour en modifier le comportement tout en utilisant le même nom de méthode. Cela fait partie du polymorphisme.

Constructeur

Un constructeur est une méthode spéciale appelée lors de la création d’un objet. Elle sert généralement à initialiser les propriétés de l’objet. En PHP, il est défini par __construct().

Gestion des exceptions

La gestion des exceptions permet de gérer les erreurs en capturant et en réagissant aux anomalies du programme via des blocs try…​catch. Cela évite que le programme ne plante en cas d’erreur inattendue.

Visibilité (public, private, protected)

La visibilité détermine où les propriétés et méthodes d’une classe peuvent être accédées :

- Public : accessible de n’importe où.
- Private : accessible uniquement depuis la classe elle-même.
- Protected : accessible depuis la classe et ses sous-classes.

Late Static Binding

Le late static binding en PHP utilise le mot-clé static:: pour s’assurer que les méthodes et propriétés sont résolues dynamiquement en fonction de la classe appelée (et non de la classe où la méthode a été définie).
